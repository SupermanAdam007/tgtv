Speech-to-text (OpenRouter)

Records from the TV microphone and transcribes via openrouter.ai/api/v1/audio/transcriptions. Model list shows approximate cost per minute (Whisper = exact ×60; GPT‑4o transcribe = illustrative token estimate — see note). Multimodal chat models filtered by audio on the models page use chat completions, not this STT endpoint. Use a short clip to stay under provider timeouts. No streaming: this API returns one full transcript after you stop recording — partial/live captions need a realtime STT product (WebSocket), not OpenRouter batch transcribe.

Whisper lines use OpenRouter list $/s × 60 → $/min ($0.006, $0.04, $0.111 per second). GPT‑4o transcribe lines are token-priced; ~$/min uses an illustrative mix (~3000 prompt + ~200 completion tokens per minute of speech). Real spend is in usage.cost after each request. GPT‑4o Mini Transcribe may return empty text for some WebM captures while still billing — if that happens, switch to GPT‑4o Transcribe or a Whisper model.

Key and model can be stored in localStorage on this device (Save). A default API key is embedded for testing — replace or remove before distribution. Many TVs need a USB mic or built-in mic support; the simulator may not provide audio input. With npm run log, each STT step emits [stt-openrouter] lines to debug.log (request meta, blob size, HTTP status, usage — never the API key or raw audio). Replay uses <audio>, then <video>, then a Web Audio route to the TV speakers. Unmute TV/system volume. The webOS TV 6 simulator often sends no sound to your Mac even when decode succeeds — test on real hardware or webOS 22+ simulator to hear playback.
